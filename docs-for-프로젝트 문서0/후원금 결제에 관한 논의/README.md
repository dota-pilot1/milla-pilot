# 후원금 결제에 관한 논의

## 문서 목록

- `index.html`: 후원금 결제, 포인트, 시설별 후원금 풀 모델 설계 논의 문서
- `erd.mmd`: 시설 후원금 풀과 포인트 원장 중심 ERD

## 핵심 결론

현실적인 MVP 순서는 `자체 물류센터 기반 직접 구매 후원`으로 시작하고, 이후 `공동 현금 후원`, `자체 쇼핑몰`, `하이브리드`, `물류 자동화`로 확장하는 것이다.

MVP 1차에서는 후원자가 외부 쇼핑몰에서 직접 물품을 구매하고 영수증/송장을 첨부한다. 플랫폼은 자체 물류센터에서 입고, 검수, 분류, 시설 전달, 웹 공유를 담당한다.

MVP 2차에서는 고가 물품을 대상으로 단체 현금 후원을 받아 플랫폼 관리자가 공동 구매하고 시설에 전달한다. MVP 3차부터는 자체 쇼핑몰로 전환해 후원자가 플랫폼 안에서 물품을 구매하고, MVP 4차에서는 자체 쇼핑몰과 공동 구매를 함께 지원한다. MVP 5차에서는 물류센터 관리 자동화와 배송 추적 시스템을 확장한다.

이 순서는 처음부터 결제/포인트/환불/정산 시스템을 크게 만들지 않고, 실제 물품 후원 수요와 입고/검수/전달 운영 가능성을 먼저 검증할 수 있다는 장점이 있다.
