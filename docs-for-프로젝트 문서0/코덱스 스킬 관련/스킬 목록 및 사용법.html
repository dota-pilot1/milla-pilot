Skill inventory Usage guide 작성일 2026-07-20

스킬 목록 및 사용법

Codex는 요청 내용에 맞는 skill을 자동으로 선택할 수 있고, 사용자가 이름을 직접 적어 호출할 수도 있다. 이 문서는 현재 환경에서 자주 쓰는 skill과 실제 요청 문장 예시를 정리한다.

실행 방법

자동 선택 “Donation Admin Tauri 앱 v0.1.6 배포해줘”처럼 업무를 자연어로 말하면 Codex가 관련 skill을 고른다.
명시 호출 tauri-app-release 스킬로 v0.1.6 릴리즈해줘처럼 skill 이름을 직접 적는다.
목록 확인 Codex CLI에서는 /skills로 skill을 탐색하고 사용할 수 있다.
명시 호출이 가장 안전하다. 특히 배포, 릴리즈, 외부 시스템 등록처럼 절차 실수가 큰 작업은 skill 이름을 함께 적는 것이 좋다.

자주 쓰는 요청 문장

# Tauri 릴리즈 tauri-app-release 스킬로 Donation Admin v0.1.6 릴리즈해줘 # PMS 등록 pms-task-register 스킬로 이 PLAN.md 내용을 Towercrane에 등록해줘 # 코드 리뷰 등록 pms-task-review 스킬로 현재 커밋 리뷰 결과를 등록해줘 # 새 skill 생성 skill-creator로 "관리앱 릴리즈 검증" skill 만들어줘 # 공식 Codex 문서 기준 확인 openai-docs 스킬로 Codex skill과 plugin 차이를 설명해줘

Skill을 고를 때 기준

  • 한 번만 하는 단순 질문이면 skill 없이 자연어로 요청해도 된다.
  • 반복 절차, 검증 기준, 외부 시스템 연동이 있으면 skill을 쓰는 것이 낫다.
  • 특정 skill을 반드시 쓰고 싶으면 skill 이름을 요청에 직접 포함한다.
  • scheduled task처럼 자동 반복 작업에서는 prompt에 $skill-name 또는 skill 이름을 명시하는 편이 안전하다.

skill은 실행 권한 그 자체가 아니다. 실제 파일 수정, 네트워크 접근, 외부 repo push 등은 현재 Codex 권한과 연결된 도구 상태에 따라 가능 여부가 결정된다.
