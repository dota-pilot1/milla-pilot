Codex customization Reusable workflow 작성일 2026-07-20

코덱스 스킬이란

Codex skill은 반복되는 작업 방식을 하나의 재사용 가능한 지침 패키지로 만든 것이다. 매번 긴 절차를 다시 설명하지 않고, 특정 업무를 요청하면 Codex가 해당 SKILL.md를 읽고 정해진 절차, 검증 방식, 참고 자료를 따라 작업한다.

한 줄 정의

스킬은 “이런 종류의 일을 할 때는 이 순서와 기준으로 처리하라”는 Codex용 실행 매뉴얼이다.

언제 쓰는가

반복 절차가 있을 때 릴리즈, 배포, 코드 리뷰 등록, 문서 생성처럼 매번 같은 체크리스트가 필요한 업무에 적합하다.
참고 자료가 정해져 있을 때 프로젝트 runbook, API 문서, 템플릿, 검증 스크립트를 함께 묶어 실수를 줄인다.
도구 사용 순서가 중요할 때 GitHub CLI, 배포 스크립트, 문서 렌더링, 외부 connector 호출 순서를 명확히 고정한다.
팀 표준을 재사용할 때 사람마다 다른 설명 없이 같은 품질 기준과 완료 기준을 반복 적용한다.

다른 Codex 설정과의 차이

구분 역할 예시
AGENTS.md repo 또는 하위 폴더에서 항상 적용되는 지속 규칙 코딩 스타일, 테스트 명령, 브랜치 정책
Skill 특정 업무를 할 때 호출되는 재사용 workflow Tauri 앱 릴리즈, PMS task 등록, PR 리뷰 처리
Plugin skill, MCP tool, app connector 등을 묶어 설치/배포하는 패키지 GitHub plugin, Google Drive plugin, 자체 업무 plugin
MCP / App connector 외부 시스템 데이터를 읽거나 실제 작업을 수행하는 도구 연결 GitHub PR 조회, Google Docs 편집, Slack 검색
Automation 정해진 시간이나 조건에 따라 반복 실행되는 작업 매일 배포 상태 확인, PR 상태 모니터링
기준은 범위다. 항상 적용되어야 하면 AGENTS.md, 특정 업무에만 필요한 절차면 skill, 외부 시스템 실행 능력이 필요하면 plugin 또는 MCP connector가 맞다.

기본 구조

개인 skill은 보통 ~/.codex/skills/{skill-name}/SKILL.md에 둔다.

--- name: tauri-app-release description: Donation Admin Tauri 앱 릴리즈 절차를 수행할 때 사용한다. --- # Tauri App Release ## Preflight - 현재 git 상태를 확인한다. - 버전 파일이 일치하는지 본다. ## Release - release repo로 소스를 동기화한다. - 태그를 푸시한다. - GitHub Actions와 latest.json을 확인한다.

이 프로젝트에서의 활용

  • tauri-app-release: Donation Admin Tauri 앱을 release repo로 동기화하고 GitHub Release를 검증한다.
  • pms-task-register: 구현 계획이나 task 문서를 Towercrane PMS에 등록한다.
  • pms-task-review: 코드 리뷰 결과를 PMS/Towercrane에 등록하거나 갱신한다.

출처 기준
