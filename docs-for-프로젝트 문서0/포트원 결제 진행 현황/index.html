포트원 결제 연동 진행 현황

milla-pilot · 후원 물품 공동구매 · 최종 업데이트 2026-07-21

한 줄 요약 테스트 채널 연동 완료

카카오페이 테스트 채널로 결제 → 서버 금액 검증 → 후원 확정 → 목표 달성 잠금까지 전 구간이 실제로 동작함을 확인했다. 실연동(실제 결제)은 아직 검증하지 않았고, 웹훅이 없다.

결제 흐름

① POST /api/contributions/payments/initiate
   → PENDING 생성 + 잔여 금액 선점(15분) + paymentId 발급

② PortOne.requestPayment(...)   ← 브라우저 SDK, 카카오페이 결제창

③ POST /api/contributions/payments/confirm
   → 서버가 포트원에 직접 조회 → 상태·금액 대조 → PAID 확정 → 모금액 반영
   → 목표 달성 시 물품 LOCKED

③이 핵심. 프론트의 "결제 성공했어요"는 위조 가능하므로 믿지 않는다. 서버가 포트원에 물어본 금액이 신청 금액과 같을 때만 확정한다.

완료된 것

백엔드

  • ContributionStatus — RECORDED / PENDING / PAID / FAILED / CANCELED / EXPIRED / REFUNDED
  • Contribution — paymentId(UNIQUE·멱등성 키) · paidAt · expiresAt · receiptUrl · pgProvider
  • PortOneClient — 토큰 교환 인증(POST /login/api-secret), 토큰 25분 캐시
  • initiate / confirm / cancelPending / 만료 스케줄러(1분 주기)
  • 동시성 — 물품 행 비관적 락 + 잔여 = 목표 − 모금액 − 선점액 으로 초과 모금 차단
  • 참여 현황 공개 API — 후원자 이름은 서버에서 마스킹(홍*동)

프론트

  • 3단계 결제 흐름 + 확정 폴링(SDK 응답이 안 올 때 대비) + 리다이렉트 복귀 처리(모바일)
  • 결제 대기 화면 — "결제를 완료했어요" 즉시 확인 / 취소 시 선점 즉시 해제
  • 결제 완료 화면 — 금액 · 목표까지 남은 금액 · 진행률 · 영수증 링크

검증 결과

실결제 건수6건 × 30,000원 = 180,000원 (테스트 채널)
물품 상태목표 600,000 / 모금 600,000 → LOCKED 전이 확인
정합성PAID 합계 + 시드(RECORDED) = 물품 raisedAmount 일치
영수증6건 전부 receiptUrl 저장됨
금액 검증포트원 조회 금액과 신청 금액 대조 후에만 확정

알려진 한계 실결제 전 해결 필요

  • 웹훅 없음 — 결제 직후 브라우저를 닫으면 확정되지 않고 15분 뒤 만료된다. 돈은 빠져나갔는데 후원은 실패하는 케이스. 폴링·리다이렉트 처리로 상당 부분 막았으나 완전하지 않다. 로컬 수신에는 ngrok 등 터널 필요.
  • 실연동 미검증 — 테스트는 목업 환경이라 앱 전환·복귀 흐름이 실제와 다르다. 출시 전 실연동 채널로 소액 결제를 반드시 한 번 해봐야 한다.
  • 영수증 링크가 테스트에서는 열리지 않음 — mockup-pg-web.kakao.com 은 실재하지 않는 도메인(DNS_PROBE_FINISHED_NXDOMAIN). 실연동에서는 정상 URL이 내려온다. 정상 동작.
  • 확정 시 실패 사유가 저장되지 않음 — 예외를 던지면 트랜잭션이 롤백되어 failReason 기록도 함께 사라진다. 운영 진단이 필요해지면 별도 트랜잭션으로 분리해야 한다.

연동 중 겪은 문제 (재발 방지 기록)

증상원인해결
RestClient$Builder 빈 없음 — 서버 기동 실패 Spring Boot 4 에서 해당 자동 구성이 별도 모듈로 분리됨 RestClient.builder() 정적 팩토리로 직접 생성
contributions_status_check 위반 Hibernate 가 만든 enum 체크 제약이 갱신되지 않음 제약 DROP 후 새 값들로 재생성
포트원 401 Invalid API secret 시크릿 위치 혼동(웹훅 시크릿 ≠ API Secret) + 인증 방식 API Keys 탭의 V2 시크릿 사용 + 토큰 교환 방식으로 전환
결제 조회 응답 역직렬화 실패 Boot 4 는 Jackson 3(tools.jackson)인데 Jackson 2 JsonNode 사용 응답을 Map<String,Object> 로 읽어 버전 의존 제거
토큰 만료 경계에서 간헐적 401 (예상) 토큰 유효기간(30분)과 캐시 기간을 동일하게 설정 캐시를 25분으로 단축
완료 화면 진행률이 100% 로 잘못 표시 캐시 갱신으로 이미 반영된 모금액에 후원액을 또 더함 갱신된 값을 그대로 사용
