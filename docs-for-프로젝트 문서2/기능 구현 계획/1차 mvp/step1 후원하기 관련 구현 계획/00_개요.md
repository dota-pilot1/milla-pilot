# Step1 후원하기 관련 구현 계획

## 목적

후원하기 MVP에서 필요한 시설 관리, 후원 물품 관리, 후원자 웹 화면, DB/ERD 설계를 한 폴더에서 추적한다.

## 폴더 구조

- `01_시설_관리_CRUD/`: Tauri 관리자 앱의 시설 등록·수정·삭제·정렬 관리 계획
- `02_후원하기_웹_페이지/`: 후원자 웹 화면, 시설 목록, 시설 상세, 물품 후원 화면 계획
- `03_DB_설계_ERD/`: 시설·후원 물품 중심 DB 설계와 ERD
- `archive/`: FAQ, 리뷰, 비교 시안 등 현재 구현 기준 문서는 아니지만 참고 가치가 있는 자료

## 관련 정책 문서

- `docs-for-프로젝트 문서/기획 문서/후원 관련 정책 요약/핵심 정책 요약.md`

## 현재 구현 기준

- 시설 이미지는 MVP에서 노출하지 않고 카드 이니셜을 사용한다.
- 후원 물품은 MVP에서 이모지 썸네일을 사용하되, 추후 이미지 필드 도입 가능성을 열어둔다.
- 상태 라벨은 `모집중`, `통합 구매 진행`, `배송중`, `목표달성`을 기준으로 통일한다.
- 관리자 화면의 시설 순서 변경은 드래그 앤 드롭 기준으로 관리한다.
