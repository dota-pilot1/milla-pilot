비밀번호 찾기 구현 계획

로그인 폼에 이메일로 비밀번호 재설정을 추가한다. beauty-book-hair(com.cj.beautybook)가 milla(com.cj.donationplatform)와 동일 구조라 백엔드는 거의 드롭인 참고가 되고, milla에 없는 건 이메일 전송 인프라(SMTP) 하나뿐이다.

1. 결론 — 가능한가?

가능하다. beauty-book에 PasswordResetToken·PasswordResetService·AuthController 엔드포인트·EmailNotificationService가 완비돼 있고, 패키지·엔티티·시큐리티 구조가 milla와 같다. 코드 이식 + SMTP 설정만 추가하면 된다.

참고 원본: beauty-book-hair/beauty-book-server/.../user/application/PasswordResetService.java, notification/application/EmailNotificationService.java, auth/domain/PasswordResetToken.java

2. milla 현재 상태 vs 필요한 것

항목milla 현재필요
Auth 엔드포인트signup·login·refresh·logout·me·check-email+ password-reset 요청/확인
이메일 전송❌ 없음 (starter-mail·JavaMailSender·spring.mail 전무)+ SMTP 인프라 신규
User 도메인email·passwordHash 있음, changePassword 없음+ changePassword()
재설정 토큰❌ 없음+ PasswordResetToken 엔티티·repo
프론트LoginForm·SignupForm (정적 export SPA)+ 찾기 링크·페이지 2개

3. 방식 선택 — 토큰 링크 (권장)

beauty-book엔 두 방식이 다 있다. milla 웹(정적 SPA)엔 토큰 링크 방식이 표준·간단.

방식흐름비고
토큰 링크 (권장)이메일로 /reset-password?token=… 링크 발송 → 클릭 → 새 비번 입력웹 표준. 정적 페이지가 쿼리 토큰 읽어 처리(§6)
인증 코드이메일로 6자리 코드 발송 → 코드 입력 → 새 비번앱/모바일 친화. 지금은 불필요

4. 흐름

[로그인 폼] "비밀번호를 잊으셨나요?" 링크
      ↓
[찾기 페이지] 이메일 입력 → POST /api/auth/password-reset/request { email }
      ↓  (서버) PasswordResetToken 생성(단일사용·만료) → 재설정 링크 메일 발송
[이메일] "비밀번호 재설정 안내" → https://dxline-tallent.com/reset-password?token=XXXX
      ↓
[재설정 페이지] 새 비번 입력 → POST /api/auth/password-reset/confirm { token, newPassword }
      ↓  (서버) 토큰 검증(만료/사용) → user.changePassword() → 토큰 markUsed
[완료] 로그인 페이지로

6. 이메일 인프라 (유일한 신규 인프라)

milla엔 메일 전송이 없다. spring-boot-starter-mail 추가 + SMTP 설정 필요.

옵션내용추천
Gmail SMTP앱 비밀번호 발급 → smtp.gmail.com:587. 설정 간단MVP·쇼케이스
AWS SES같은 계정(ap-northeast-2), 도메인 검증 필요. 발송량·신뢰성↑프로덕션
 
build.gradle:  implementation 'org.springframework.boot:spring-boot-starter-mail'

application.yaml:
  spring.mail:
    host: ${MAIL_HOST:smtp.gmail.com}
    port: ${MAIL_PORT:587}
    username: ${MAIL_USERNAME:}
    password: ${MAIL_PASSWORD:}          # Gmail 앱 비밀번호
    properties.mail.smtp: { auth: true, starttls.enable: true }
  app:
    front-url: ${APP_FRONT_URL:https://dxline-tallent.com}
배포 반영: EC2 /home/ubuntu/milla/.env에 MAIL_USERNAME·MAIL_PASSWORD·APP_FRONT_URL 추가 후 milla-server 재시작. (비밀값은 .env에만)

7. 프론트 구현 (정적 export 호환)

  • 로그인 폼(features/auth/login/LoginForm.tsx): "비밀번호를 잊으셨나요?" 링크 → /forgot-password
  • 찾기 페이지 /app/forgot-password/page.tsx: 이메일 입력 → request 호출 → "메일을 확인하세요" 안내
  • 재설정 페이지 /app/reset-password/page.tsx: useSearchParams().get("token")로 토큰 읽어 새 비번 → confirm 호출 (step 프론트 정적화와 동일 패턴)
정적 export라 reset-password?token=는 쿼리스트링으로 받는다(경로 파라미터 X) — /donate/facility?id= 전환과 같은 방식. useSearchParams는 <Suspense>로 감싼다.

8. 보안 고려

  • 토큰: UUID·단일사용(used)·만료(예 30분). 사용/만료 시 거부.
  • 유저 열거 방지: 이메일이 없어도 항상 200 반환 권장(가입 여부 노출 금지). beauty-book은 USER_NOT_FOUND를 던지므로 milla에선 "성공 응답 고정"으로 개선.
  • 레이트 리밋: 동일 이메일 재요청 시 기존 토큰 무효화(deleteByUserId) + 짧은 재발송 간격.
  • 메일 실패 격리: 발송 실패해도 요청 트랜잭션은 성공 처리(로그만).

9. 범위 / 체크리스트

이번(Step7)

  • 토큰 링크 방식 비밀번호 재설정(요청→메일→확인)
  • SMTP 메일 인프라 + 재설정/변경완료 HTML 메일
  • 로그인 폼 링크 + 찾기/재설정 페이지 2개

범위 밖(차기)

  • 인증 코드 방식·SMS·소셜 로그인 복구
  • 비밀번호 정책 강화(재사용 금지 등)

구현 순서

1) build.gradle starter-mail + application.yaml spring.mail/app.front-url
2) PasswordResetToken 엔티티·repo
3) EmailNotificationService (JavaMailSender)
4) PasswordResetService + User.changePassword + ErrorCode
5) AuthController 2 엔드포인트 + SecurityConfig permitAll
6) 프론트 forgot-password / reset-password 페이지 + 로그인 링크
7) EC2 .env(MAIL_*·APP_FRONT_URL) + 재배포 + 실제 메일 수신 검증
