디자인 시스템 & 인프라 구현

깔끔·모던한 웹을 일관되게 찍어내기 위한 두 축 — 디자인 컨셉(톤·색·간격)과 인프라 구현(문서 거버넌스·토큰·공통 컴포넌트·FSD 배치). 기준 원본은 /docs/design-system.md, 이 문서는 그 개요를 시각화한 보기용이다.

1. 디자인 컨셉

후원 이행 플랫폼. 감성 과잉 없이 신뢰·정확·차분. shadcn/ui new-york 톤을 기본으로 한다.

  • 시맨틱 토큰만 사용 — raw 팔레트(bg-emerald-500 등) 금지. 테마 전환(라이트/브랜드 hue) 안전.
  • 둥근 카드(rounded-xl) + 얇은 보더 + 은은한 그림자. 강한 장식 지양.
  • 아이콘 lucide-react. 진행률·상태는 데이터 기반으로만 표시.
  • 아동 정보 0노출 · "100% 전달" 등 보장 문구 금지(기획 제약).

7. primitive 규약

  • variant는 class-variance-authority(cva), className 병합은 cn()(@/shared/lib/utils = twMerge(clsx())).
  • variant / size prop 노출 + className 오버라이드 허용.
  • 폼 요소는 disabled · focus-visible:ring-ring · aria-* 포함.
// 예: Button
const buttonVariants = cva("inline-flex ... focus-visible:ring-ring", {
  variants: { variant: { default: "bg-primary text-primary-foreground ...", ... },
              size: { default: "h-10 px-4", sm: "h-9 px-3", ... } },
  defaultVariants: { variant: "default", size: "default" },
});
<button className={cn(buttonVariants({ variant, size }), className)} />

8. 상태 표시 컨벤션

  • 진행률 = raised / goal 실제 금액 기준. 추상 목표 금지.
  • 상태 구분색은 shared/ui/Badge 한 파일에서만 palette 사용(단일 출처). 페이지·피처에서 상태색 직접 칠하기 금지.
  • 로딩/빈 상태는 EmptyState/스켈레톤으로 통일.
상태색을 Badge 한 곳에 몰아두면, 나중에 색을 바꿔도 한 파일만 고치면 되고 화면 전체가 일관되게 따라온다 — 테마 드리프트 방지의 핵심.
