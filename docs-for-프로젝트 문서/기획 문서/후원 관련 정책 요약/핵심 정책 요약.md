# 후원 관련 핵심 정책 요약

> 1차 MVP 기준 정책 요약이다. 구현 문서와 UI 문구는 이 문서를 우선 참고한다.

## 1. 시설 관련

- 시설은 후원 대상 단위이며, 후원 물품을 소유하는 상위 엔티티다.
- 시설 소개에는 아동 개인정보를 포함하지 않는다.
- 시설명, 지역, 시설 유형, 자격확인 여부, 표시 상태 정도만 노출한다.
- MVP에서는 시설별 이미지를 사용하지 않는다.
- 시설 카드에는 `avatar_initial`을 사용한다. 예: `햇`, `푸`, `온`.
- 시설 이미지는 정식 운영 전 별도 정책 수립 후 검토한다.
- 실제 시설 사진, 아동 사진, 내부 공간 사진, 간판 등 식별 가능한 이미지는 기본적으로 사용하지 않는다.
- 시설 이미지를 도입하더라도 검수된 대표 이미지 또는 추상 커버 이미지 중심으로 검토한다.
- 시설 삭제 시 소속 후원 물품도 함께 정리되는 것을 전제로 한다.

## 2. 후원 물품 관련

- 후원 물품은 시설별 준비물 목표다.
- 물품명, 카테고리, 수량 메모, 필요한 이유, 목표금액, 모인금액, 상태, 마감일을 관리한다.
- 목표금액은 실제 구매 총액 기준으로 표시한다.
- 모인금액은 MVP에서는 표시용이며, 차기 단계에서 후원 참여 합계로 대체한다.
- MVP에서는 물품 썸네일로 `emoji`를 사용한다.
- `emoji`는 강제 정책이 아니라 이미지 업로드 전 단계의 가벼운 fallback 표현이다.
- 정식 운영 전에는 `thumbnailImageUrl` 같은 물품 대표 이미지 필드 도입을 검토한다.
- 물품 이미지는 시설 이미지보다 우선 검토할 수 있다. 단, 저작권과 검수 기준이 필요하다.
- 이미지가 없는 경우 카테고리 기본 이미지 또는 이모지 fallback을 사용한다.
- 상태 표시는 배지로 통일한다. 예: 모집중, 목표달성, 통합 구매 진행, 배송중.

## 3. 그밖의 정책

- "100% 전달"처럼 법적·회계적으로 검증되지 않은 보장 문구를 쓰지 않는다.
- 후원금은 시설 후원금 전용계좌로 직접 귀속되는 구조를 전제로 설명한다.
- 결제 기능은 전문가 검토 후 별도 단계에서 도입한다.
- 아동·청소년 사진, 실명, 얼굴, 생활공간, 위치 등 식별 가능 정보 노출을 피한다.
- 아동 관련 이미지를 사용할 경우 아동 및 보호자/법정대리인 동의, 사용 목적, 공개 범위, 철회 방법을 별도 정책으로 정해야 한다.
- 후원자 화면은 관리 용어보다 사용자 행동 언어를 사용한다. 예: 관리보다 보기, 후원하기, 내역.
- 관리자 화면은 반복 업무 중심의 운영 콘솔로 만든다.
- 디자인보다 신뢰, 정보 명확성, 개인정보 보호, 운영 가능성을 우선한다.
- MVP에서는 기능 범위를 작게 유지하고, 이미지·결제·영수증·구매 이행 추적은 단계적으로 확장한다.
