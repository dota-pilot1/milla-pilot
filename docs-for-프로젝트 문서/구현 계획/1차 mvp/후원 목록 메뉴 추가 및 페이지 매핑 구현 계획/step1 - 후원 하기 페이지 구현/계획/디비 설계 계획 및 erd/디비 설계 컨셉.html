Step 1 DB 설계 Spring · JPA 2 tables

DB 설계 컨셉 — 시설물 · 후원 물품

후원하기 → 시설물 목록 → 물품 목록 을 지탱하는 최소 스키마. 테이블 2개(facilities 1 : N donation_items)로 시작하고, 결제·후원참여 테이블은 다음 스텝으로 분리한다. 기존 role·menu 도메인의 JPA 컨벤션을 그대로 따른다.

1. 관계 한눈에

facilities 1 ──< N donation_items 시설 하나에 후원 물품 여러 개
한 시설이 여러 준비물(물품) 목표를 가진다. 시설 삭제 시 물품도 함께 삭제 (ON DELETE CASCADE / JPA orphanRemoval).
ERD 다이어그램 소스는 같은 폴더의 erd.mmd (Mermaid). 아래 6절에 소스 첨부.

컬럼타입제약설명

3. donation_items — 후원 구매 물품

컬럼타입제약설명
idBIGINTPK IDENTITY기본키
facility_idBIGINTFK NOT NULL@ManyToOne → facilities
nameVARCHAR(150)NOT NULL물품명
categoryVARCHAR(30)ENUMLEARNING / LIVING / HYGIENE
noteVARCHAR(150)수량 메모 (예: 6종·30권)
reasonVARCHAR(1000)필요한 이유 (물품 상세용)
emojiVARCHAR(10)썸네일 이모지
goal_amountBIGINTNOT NULL목표금액(원) — 실제 구매 총액 기준
raised_amountBIGINTDEFAULT 0모인금액 — 이번엔 표시용, 차기 Σ CONTRIBUTION
target_quantityINT목표수량
statusVARCHAR(20)ENUMRECRUITING / LOCKED / BUYING / SHIPPING
deadlineDATE마감일 (nullable)
display_orderINTDEFAULT 0정렬 순서
created_atTIMESTAMPNOT NULL@CreationTimestamp
updated_atTIMESTAMPNOT NULL@UpdateTimestamp
raised_amount 주의. 이번 스텝에서는 관리자가 넣거나 0으로 두는 표시용 값이다. 실제 결제 스텝에서 CONTRIBUTION 테이블이 생기면 이 값은 후원 참여액의 합계로 대체된다. 진행률 = raised_amount / goal_amount (추상 모금 목표 금지, 기획안 F-013).

4. Enum 값 정리

Enum값용도
FacilityTypeORPHANAGE, CHILD_CENTER아동양육시설 / 지역아동센터
FacilityStatusRECRUITING, BUYING, SHIPPING시설 카드 상태 배지
ItemCategoryLEARNING, LIVING, HYGIENE학습 / 생활 / 위생 — 필터
ItemStatusRECRUITING, LOCKED, BUYING, SHIPPING물품 상태 — 목표달성 시 LOCKED
JPA는 @Enumerated(EnumType.STRING) 로 문자열 저장 (순서 변경에 안전).

5. 이번 범위 / 차기 분리

✅ 이번 스텝 (실선 테이블)

  • facilities, donation_items — CRUD(Tauri) + 조회(웹)

⛔ 다음 스텝 (결제 도입 시 추가)

  • contributions — 후원자별 참여 기록(기부 원장). raised_amount를 대체.
  • purchase_orders — 목표달성 후 통합구매·배송 추적.
지금 두 테이블만 잘 만들면 차기 테이블은 donation_item_id FK로 자연스럽게 붙는다. 그래서 donation_items가 향후 후원·구매의 연결 허브가 된다.

같은 폴더 erd.mmd 와 동일. Mermaid Live 등에 붙여 렌더.

7. JPA 매핑 노트 (기존 컨벤션)

  • @Entity + @Table(name="...", uniqueConstraints=...)
  • @Id @GeneratedValue(strategy = IDENTITY) Long id
  • @NoArgsConstructor(access = PROTECTED) + 정적 팩토리 create(...), 도메인 메서드로 변경
  • @CreationTimestamp / @UpdateTimestamp (Instant)
  • FK: @ManyToOne(fetch = LAZY) @JoinColumn(name = "facility_id")
  • 패키지: com.cj.donationplatform.facility.*, ...donation_item.*
