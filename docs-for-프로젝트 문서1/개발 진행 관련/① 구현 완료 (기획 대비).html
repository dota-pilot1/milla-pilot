구현 완료 (기획 대비)

기획 문서에서 요구한 것 중 지금 코드로 동작하는 것만 근거(파일·엔드포인트)와 함께 정리한다. 이행 루프 본체 6개 + 정합성 안전장치 + 구매·배송·수령 확장 + 정책 필드 + 공통 기반까지 완료. 아직 안 된 것은 별도 문서 ② 미구현 · 구현 예정 (기획 대비).html 참고.

1. 이행 루프 본체 6개 (웹 3 · 타우리 3)

근거: 제품 핵심 기능 및 범위.html §5·§8 — "본체 완성 기준: 6/6"

기능면상태구현 근거
후원하기 — 시설·물품 탐색, 공동충당 참여웹✅ 완료/donate, /donate/facilities/[id], /donate/items/[id] · features/donate · POST /api/contributions
후원 내역 — 내 기여·진행단계, 물품별 합산웹✅ 완료/my-donations · features/my-donations · GET /api/contributions/me
물품 구매 현황 — 구매·증빙·송장 딥링크웹✅ 완료PurchaseStatusCard · GET /api/donation-items/{id}/purchase-order (공개)
시설 관리 — 시설·준비물 CRUD, 단계 전환타우리✅ 완료features/facility-management · FacilityController · DonationItemController
후원 내역(기부 원장) — 물품별·전체 조회타우리✅ 완료features/contribution-ledger · GET /api/contributions, /by-item/{id}
통합 구매 — 실행→완료, 증빙·송장 기록타우리✅ 완료features/purchase-management · PurchaseOrderController
본체 6/6 완료. 시설 준비물 목표 → 공동충당 → 목표달성 잠금 → 통합구매 → 배송조회 → 수령확인까지 양쪽(웹·타우리)에서 닫힌다. (마지막 "영수증 발급"만 미완 — 문서②)

2. 정합성 · 잠금 (핵심 안전장치)

근거: step3 후원 목표 금액 초과시 정합성 동시성 문제 처리 · §15 KPI "중복구매 0"

항목상태구현 근거
목표달성 시 자동 잠금(LOCKED) — 추가결제 차단✅ContributionService · item.addRaised() 후 도달 시 markLocked()
초과 충당 방지 + 동시성(비관적 락)✅findByIdForUpdate (@Lock PESSIMISTIC_WRITE) — 20건 동시요청 검증 완료
중복구매 0 — 물품당 통합구매 1건✅purchase_orders.donation_item_id UNIQUE 제약

3. 구매 → 배송 → 수령 확장 (step5 · step6 Phase1)

항목상태내용 · 근거
통합구매 기록✅판매처, 판매처 URL, 구매일, 구매액, 구매수량, 주문/영수증번호, 영수증 이미지 첨부, 메모 — PurchaseOrder + S3 presign
송장(딥링크)✅택배사·송장번호·송장이미지·배송예정일 · tracker.delivery 통합조회 딥링크 · 송장번호 채우면 자동 배송중(SHIPPING)
수령확인✅사람 1클릭 → RECEIVED · POST /api/donation-items/{id}/receive (배송완료 ≠ 수령확인)
두 입력 경로 유지✅구매 폼에서 송장 함께 or 완료목록 별도 '송장 등록' 단계 — 팀 논의 여지
배송 단계 자동화(집계 API pull)는 step6 Phase2로 남김(문서②). 지금은 송장 1회 입력 + 딥링크 조회까지.

근거: 후원 관련 정책 요약/핵심 정책 요약.md

물품: 물품명·카테고리·수량메모·필요이유·목표금액·모인금액·상태·마감일·emoji✅DonationItem: name·category·note·reason·goalAmount·raisedAmount·status·deadline·emoji·targetQuantity

상태 배지: 모집중·목표달성·통합구매진행·배송중✅배지 단일 출처(Badge / ITEM_STATUS_*)
"100% 전달" 등 미검증 보장문구 미사용✅구매증빙=세액공제 영수증 아님(F-015) 명시
