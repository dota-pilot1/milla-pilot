핵심 비즈니스 로직 1차 MVP · 폐쇄형 이행관리

제품 핵심 기능 및 범위

이 제품의 본체는 "시설 준비물 목표 → 후원자 공동충당 → 목표달성 → 통합구매 → 수령·증빙·영수증 → 후원자 상태 확인"의 후원 이행 루프(기획 §5·§7)다. 웹 3 + 타우리 3, 총 6개 기능이 이 루프를 양쪽에서 구현한다.

1. 핵심 비즈니스 로직 (이행 루프)

[시설] 준비물 목표 등록·검수
   → [후원자] 공동충당 (품목별 금액 참여)
   → 목표달성 시 자동 잠금 (추가결제 차단, 중복구매 0)
   → [운영자] 통합구매 1회 (판매처·실구매액·증빙 기록)
   → 판매처 → 시설 직배송 → 시설 수령확인
   → 영수증 발급자료 → 후원자 화면에 단계 반영
이 루프가 닫히면 = 핵심 비즈니스 로직 완성. 남은 건 이 루프에 안전장치·확장을 붙이는 것(부가).

2. 핵심 기능 6개 (웹 3 · 타우리 3)

웹 — 후원자 대면

후원하기
시설·물품 탐색, 공동충당 참여		완료
후원 내역 보기
내 기여·진행 단계(타임라인), 물품별 합산		완료
물품 구매 현황
구매 진행·완료, 영수증·송장 표시 + 배송 추적(송장→드로워)		예정 · 구매(step5 후속) + 배송(step6)

타우리 — 관리자

시설 관리
시설·준비물 목표 CRUD, 단계 전환		완료
후원 내역(기부 원장)
물품별·전체 후원자 기록 조회		완료
통합 구매
구매 실행→완료 상태변경, 영수증·송장 첨부		계획 완료·구현 예정(step5)
본체 완성 기준: step5(통합구매) + 웹 물품 구매 현황 → 6/6. 현재 4/6 완료.

3. 범위 3분류

분류예성격
본체 (핵심)위 6개 기능 (이행 루프)지금 완성 중. 이게 제품의 이유
프로덕션 필수 부가실결제(PG)·취소/환불 정합성·관리자 authz 세분화·감사메시지본체 기능은 아니지만 실서비스엔 필수. 전문가 검토 관문 뒤(§5·F-016)
순수 부가 · 차기커뮤니티·명예홍보관·협력교실·통계 대시보드·i18n·펀딩캠페인·실시간 택배추적없어도 본체 작동. 파일럿 이후(§7·§10·§20)
주의: "본체 이후는 다 부가"는 맞지만, 실결제·환불·authz는 순수 부가와 달리 실서비스 전 반드시 필요. 그리고 실시간 택배추적(택배사 API)은 파일럿 이후 — MVP 구매현황은 기록된 상태 + 송장번호까지.

4. 지금 위치 & 다음

  • 완료: 웹 후원하기·후원내역 / 타우리 시설관리·기부원장 + 정합성(초과·동시성)·단계관리·authz 기초
  • 다음(본체 마무리): ① step5 통합구매(타우리) → ② 웹 물품 구매 현황(구매·증빙) → ③ step6 배송 추적(송장 1회 입력 + 자동 조회, 관리자 수동 X)
  • 그다음: 프로덕션 필수 부가(실결제·환불·authz) → 순수 부가(선택)
