Donation Admin Tauri

Tauri 앱 업데이트와 릴리즈 운영 가이드

관리자 데스크톱 앱의 업데이트 체크가 어떤 조건에서 동작하는지, 현재 실패 원인이 무엇인지, 이후 릴리즈를 어떤 순서로 만들어야 하는지 정리한 문서입니다.

작성일 2026-07-19 대상 앱 donation-admin-tauri 다음 앱 버전 0.1.2 업데이트 저장소 dota-pilot1/donation-admin-tauri

1. 결론

v0.1.1 문제는 두 가지가 겹친 상태입니다. 앱은 /releases/latest/download/latest.json을 조회하지만, v0.1.1 릴리즈가 pre-release라서 GitHub의 latest 경로에서 잡히지 않습니다. 또한 macOS job은 Apple Developer ID 인증서 secret이 없어 security import 단계에서 실패했고, 그래서 Windows 설치 파일만 업로드되었습니다.
릴리즈 목록 https://github.com/dota-pilot1/donation-admin-tauri/releases
최신 릴리즈 https://github.com/dota-pilot1/donation-admin-tauri/releases/latest
현재 릴리즈 v0.1.1 https://github.com/dota-pilot1/donation-admin-tauri/releases/tag/v0.1.1
v0.1.1 Windows 설치 파일 https://github.com/dota-pilot1/donation-admin-tauri/releases/download/v0.1.1/Donation.Admin_0.1.1_x64-setup.exe
Updater feed https://github.com/dota-pilot1/donation-admin-tauri/releases/latest/download/latest.json
항목 현재 상태 판단
앱 설정 endpoint https://github.com/dota-pilot1/donation-admin-tauri/releases/latest/download/latest.json 주의 현재 메인 repo가 아니라 별도 릴리즈 repo를 바라봅니다.
태그 직접 접근 /releases/download/v0.1.1/latest.json 접근 가능 정상 에셋 자체는 존재합니다.
latest 접근 /releases/latest/download/latest.json 404 실패 pre-release는 latest 대상이 아니어서 발생할 수 있습니다.
macOS 설치 파일 v0.1.1에는 .dmg 없음 실패 macOS job이 Apple 인증서 import 단계에서 중단되었습니다.
다음 릴리즈 기준 v0.1.2 정식 release 권장 macOS .dmg, Windows .exe, latest.json을 함께 확인합니다.

2. 업데이트 체크가 동작하는 조건

Tauri updater는 웹 화면의 새로고침과 다릅니다. 앱 내부에서 @tauri-apps/plugin-updater의 check()를 호출하고, 설정된 endpoint의 latest.json을 받아 현재 앱 버전과 비교합니다.

개발 서버에서

npm run tauri dev에서도 호출 자체는 가능하지만, 실제 운영 업데이트 검증에는 부적합합니다.

업데이트 설치는 서명된 번들, updater artifact, 릴리즈 에셋을 전제로 하기 때문입니다.

빌드된 앱에서

실제 검증 기준입니다. 설치된 앱 버전보다 높은 버전의 latest.json과 서명 파일이 있어야 업데이트가 표시됩니다.

같은 버전이면 업데이트 없음으로 처리됩니다.

const update = await check();
if (update) {
  await update.downloadAndInstall();
  await relaunch();
}

3. 현재 설정

현재 관리자 앱 설정은 다음과 같습니다.

{
  "productName": "Donation Admin",
  "version": "0.1.2",
  "bundle": {
    "targets": ["app", "dmg", "nsis"],
    "createUpdaterArtifacts": true
  },
  "plugins": {
    "updater": {
      "endpoints": [
        "https://github.com/dota-pilot1/donation-admin-tauri/releases/latest/download/latest.json"
      ]
    }
  }
}
중요한 점 소스 코드는 dota-pilot1/milla-pilot에 있고, 앱 업데이트 endpoint는 dota-pilot1/donation-admin-tauri를 바라봅니다. 릴리즈를 어느 저장소에서 운영할지 먼저 고정해야 합니다.

4. 정상 릴리즈 절차

  1. 릴리즈할 변경사항을 milla-pilot에 커밋하고 main에 푸시합니다.
  2. CI와 서버/프론트 배포가 성공했는지 확인합니다.
  3. donation-admin-tauri/src-tauri/tauri.conf.json의 version을 올립니다. 예: 0.1.2
  4. 서명 환경변수를 준비합니다. updater에는 TAURI_SIGNING_*, macOS 서명/공증에는 APPLE_* secret이 필요합니다.
  5. donation-admin-tauri에서 Tauri 앱을 빌드합니다.
  6. 생성된 macOS .dmg, Windows .exe, 서명 파일, latest.json을 업데이트 저장소 릴리즈에 업로드합니다.
  7. 릴리즈를 pre-release가 아닌 정식 release로 게시합니다.
  8. 설치된 이전 버전 앱에서 업데이트 체크를 실행해 새 버전이 감지되는지 확인합니다.
cd donation-admin-tauri
npm run tauri build
주의 현재 v0.1.1을 정식 release로 바꾸는 것만으로는 이번 커밋의 데스크톱 앱이 배포되지 않습니다. 앱 버전을 올리고 새 산출물을 만들어야 합니다.
Apple secret 기준 macOS job에는 APPLE_CERTIFICATE, APPLE_CERTIFICATE_PASSWORD, APPLE_SIGNING_IDENTITY, APPLE_ID, APPLE_PASSWORD, APPLE_TEAM_ID가 모두 필요합니다. Donation Admin repo에는 2026-07-19에 기존 배포 앱과 같은 Developer ID secret 세트를 재등록했습니다.

5. 저장소 운영 선택지

선택지 장점 주의점
현재처럼 donation-admin-tauri 별도 저장소에 릴리즈 업데이트 산출물 저장소가 분리되어 깔끔합니다. 소스 커밋과 릴리즈 산출물의 출처가 달라서 어떤 커밋으로 빌드했는지 release note에 반드시 남겨야 합니다.
milla-pilot 저장소 릴리즈로 통합 소스, CI, 태그, 릴리즈가 한 저장소에 묶여 추적이 쉽습니다. tauri.conf.json의 endpoint를 바꾸고 기존 설치 앱의 업데이트 경로 전환을 고려해야 합니다.
권장 단기적으로는 현재 endpoint를 유지하되, 별도 저장소 릴리즈에 "빌드 기준 커밋 SHA"를 명시합니다. 장기적으로 자동 릴리즈 워크플로를 만들 때는 milla-pilot 통합 릴리즈가 추적성이 더 좋습니다.

6. 문제 진단 체크리스트

증상 확인 명령 의미
업데이트 확인 실패 curl -I -L https://github.com/dota-pilot1/donation-admin-tauri/releases/latest/download/latest.json 404면 latest 릴리즈 문제입니다. 서명 검증까지 가지 못한 상태입니다.
태그 직접 접근은 됨 curl -I -L https://github.com/dota-pilot1/donation-admin-tauri/releases/download/v0.1.1/latest.json 에셋은 있지만 latest 경로가 비어 있는 상태일 수 있습니다.
업데이트 없음 latest.json의 version과 설치 앱 버전 비교 같은 버전이면 정상적으로 업데이트 없음입니다.
다운로드 후 설치 실패 설치 파일 .sig와 pubkey 매칭 확인 이 단계부터는 서명 문제일 가능성이 큽니다.
macOS 산출물 없음 gh run view <run-id> --repo dota-pilot1/donation-admin-tauri --log-failed security import 실패면 Apple 인증서 secret 형식, 비밀번호, identity 값을 먼저 확인합니다.

7. 다음 릴리즈 권장안

  1. tauri.conf.json 버전을 0.1.2로 올립니다.
  2. 현재 donation-admin-tauri main 기준으로 Tauri 앱을 빌드합니다.
  3. v0.1.2 릴리즈를 dota-pilot1/donation-admin-tauri에 정식 release로 만듭니다.
  4. 릴리즈 노트에 macOS/Windows 설치 파일과 updater artifact가 모두 포함되었는지 남깁니다.
  5. latest.json이 /releases/latest/download/latest.json에서 200으로 내려오는지 확인합니다.
  6. 기존 0.1.1 설치 앱에서 업데이트 체크를 눌러 0.1.2가 표시되는지 확인합니다.
자동화 기준 donation-admin-tauri repo는 태그 푸시 시 GitHub Actions가 macOS/Windows를 매트릭스로 빌드합니다. prerelease: false 상태여야 앱의 updater endpoint가 최신 릴리즈를 정상 조회합니다.

참고: 2026-07-19 확인 기준, v0.1.1에는 Windows .exe, .sig, latest.json 에셋만 있고 macOS .dmg는 없습니다. v0.1.2부터 macOS와 Windows 산출물을 함께 확인합니다.
