Donation Admin Tauri

Tauri 인증·서명 Secret 운영 가이드

macOS 서명·공증, Tauri updater 서명, GitHub Actions secret 등록 기준을 정리한 문서입니다. 실제 인증 값은 Git에 올리지 않고 로컬 .local-secrets/와 GitHub repository secrets에만 둡니다.

작성일 2026-07-19 대상 repo dota-pilot1/donation-admin-tauri 릴리즈 v0.1.2 기준

1. 결론

값은 저장하되 Git에는 올리지 않습니다. 이 폴더 아래 .local-secrets/를 로컬 보관소로 사용하고, 루트 .gitignore에서 **/.local-secrets/를 제외 처리합니다. 문서에는 secret 값 자체를 쓰지 않습니다.
APPLE_* 시크릿 누락 = 미서명이 아니라 “빌드 실패”

“Apple 시크릿이 없으면 미서명으로 나온다”는 틀린 설명입니다. 워크플로에 APPLE_CERTIFICATE: ${{ secrets.APPLE_CERTIFICATE }} 줄이 있는데 시크릿이 등록돼 있지 않으면, 변수가 없는 게 아니라 빈 문자열로 주입됩니다. Tauri는 이를 “서명하라는 지시”로 해석하고 빈 인증서를 키체인에 import하려다 실패합니다. 

security: SecKeychainItemImport: One or more parameters passed to a function were not valid.
failed to bundle project: failed codesign application

결과적으로 macOS job만 죽고 Windows는 성공합니다. 릴리즈에는 .exe만 올라가고 .dmg가 없으며, latest.json에서 darwin-* 항목이 빠져 Mac 자동 업데이트가 조용히 멈춥니다.

선택지는 둘뿐이며 중간은 없습니다.

  1. 서명한다 — APPLE_* 6개를 모두 등록 (권장)
  2. 서명하지 않는다 — 워크플로에서 APPLE_* env 줄을 삭제합니다. 주석 처리나 빈 시크릿으로 두면 안 됩니다. 그래야 진짜 미서명 .dmg가 나옵니다.

실제 사고 2건 — donation-admin-tauri v0.1.1, milla-dev-task-tauri v0.1.10·v0.1.11 (2026-07-19 해소). 시크릿은 repo마다 별도이므로 앱을 새로 만들면 8개를 전부 다시 등록해야 합니다.

릴리즈 전 대조 (30초) 
# 워크플로가 요구하는 시크릿
grep -oE '[A-Z_]+: \$\{\{ secrets\.[A-Z_]+' .github/workflows/tauri-release.yml
# 실제 등록된 시크릿 (값은 출력되지 않음)
gh secret list --repo <owner>/<repo>
두 목록이 다르면 릴리즈를 시작하지 않습니다. 릴리즈 후에는 자산에 .dmg와 .exe가 모두 있는지 확인합니다.

2. Secret 종류

Secret 용도 필수 시점
TAURI_SIGNING_PRIVATE_KEY Tauri updater artifact 서명 업데이트 설치를 지원하는 모든 릴리즈
TAURI_SIGNING_PRIVATE_KEY_PASSWORD updater private key 비밀번호 private key가 비밀번호를 가질 때
APPLE_CERTIFICATE Developer ID Application .p12를 base64로 인코딩한 값 macOS 서명·공증 릴리즈
APPLE_CERTIFICATE_PASSWORD .p12 인증서 비밀번호 macOS 서명·공증 릴리즈
APPLE_SIGNING_IDENTITY 코드서명 identity. 현재 기준 Developer ID Application: Hyunseok oh (5PRM3RRTSH) macOS 서명·공증 릴리즈
APPLE_ID, APPLE_PASSWORD, APPLE_TEAM_ID Apple 공증 계정, app-specific password, team id macOS 공증 릴리즈

3. 로컬 보관 구조

Donation Admin 문서 폴더의 로컬 secret 보관 위치입니다.

docs-for-프로젝트 문서1/배포 관련/tauri 인증 관련/.local-secrets/
  apple_certificate_base64.txt
  apple_certificate_password.txt
  apple_id.txt
  apple_app_specific_password.txt
주의 TAURI_SIGNING_PRIVATE_KEY는 앱의 updater public key와 한 쌍입니다. Donation Admin의 public key는 Twillo/English 앱의 public key와 다르므로 다른 앱의 updater private key를 복사해 쓰면 안 됩니다.

4. GitHub Secrets 등록

값을 출력하지 않고 파일에서 바로 등록합니다.

cd /Users/terecal/milla-pilot
SECRETS_DIR="docs-for-프로젝트 문서1/배포 관련/tauri 인증 관련/.local-secrets"
REPO="dota-pilot1/donation-admin-tauri"

gh secret set APPLE_CERTIFICATE --repo "$REPO" < "$SECRETS_DIR/apple_certificate_base64.txt"
gh secret set APPLE_CERTIFICATE_PASSWORD --repo "$REPO" < "$SECRETS_DIR/apple_certificate_password.txt"
gh secret set APPLE_ID --repo "$REPO" < "$SECRETS_DIR/apple_id.txt"
gh secret set APPLE_PASSWORD --repo "$REPO" < "$SECRETS_DIR/apple_app_specific_password.txt"
gh secret set APPLE_SIGNING_IDENTITY --repo "$REPO" --body "Developer ID Application: Hyunseok oh (5PRM3RRTSH)"
gh secret set APPLE_TEAM_ID --repo "$REPO" --body "5PRM3RRTSH"

등록 상태 확인은 이름만 확인합니다.

gh secret list --repo dota-pilot1/donation-admin-tauri

5. 실패 진단

증상 확인 조치
security import 실패 Actions 실패 로그의 macOS job APPLE_CERTIFICATE가 비어 있거나 base64 p12가 아니거나, 비밀번호가 맞지 않는지 확인합니다.
Windows만 릴리즈됨 Release assets에 .dmg 없음 macOS job 실패 로그를 먼저 확인합니다. Windows job 성공만으로 릴리즈가 일부 생성될 수 있습니다.
Mac만 자동 업데이트 안 됨 latest.json에 darwin-* 항목 없음 macOS 빌드가 실패한 것입니다. 위 두 항목을 확인한 뒤 재릴리즈합니다.
base64 인증서 import 실패 시크릿 값에 개행이 섞임 --body "$(tr -d '\n' < apple_certificate_base64.txt)" 형태로 재등록합니다.
업데이트 확인 실패 /releases/latest/download/latest.json 404 릴리즈가 pre-release인지 확인하고, 정식 release로 게시합니다.
