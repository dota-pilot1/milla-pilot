GitHub Actions 대상 repo: dota-pilot1/milla-pilot Secrets는 하나씩 등록 비밀값 git 금지

배포 자동화 관련 GitHub Actions 변수

이 문서는 GitHub Actions에서 백엔드 EC2 배포와 프론트 S3/CloudFront 배포를 자동화할 때 등록해야 하는 Actions Secrets를 한 곳에 정리한 것이다. GitHub 화면에서는 Name과 Secret을 한 쌍씩 등록해야 하며, 여러 이름을 Secret 칸에 한꺼번에 붙여 넣으면 안 된다.

1. 등록 페이지

아래 주소에서 New repository secret을 눌러 하나씩 등록한다.

https://github.com/dota-pilot1/milla-pilot/settings/secrets/actions

입력 방식: Name에는 변수명 하나, Secret에는 그 값 하나만 넣는다. 예를 들어 EC2_HOST를 등록할 때는 Name에 EC2_HOST, Secret에 13.209.195.64만 넣는다.

2. 현재 라이브 배포 기준 Secrets

GitHub Name에 넣을 값 GitHub Secret에 넣을 값 출처 / 비고
EC2_HOST 13.209.195.64 현재 라이브 EC2 IP
EC2_USER ubuntu EC2 SSH 사용자
EC2_SSH_KEY hibot-d-server-key.pem 파일 내용 전체 파일 경로가 아니라 private key 본문 전체를 붙여 넣는다.
AWS_ACCESS_KEY_ID AWS IAM Access Key ID S3 sync + CloudFront invalidation 권한 필요
AWS_SECRET_ACCESS_KEY AWS IAM Secret Access Key 절대 문서/git에 저장하지 않는다.
AWS_REGION ap-northeast-2 서울 리전
FRONT_S3_BUCKET beauty-book-hair-front 현재 milla 정적 프론트가 올라간 S3 버킷
CLOUDFRONT_DISTRIBUTION_ID E11NF3HMOB52NI 현재 dxline-tallent.com CloudFront 배포 ID
NEXT_PUBLIC_API_URL https://dxline-tallent.com 프론트 빌드 시 주입할 운영 API base URL
현재 기준: 이 값들은 docs-for-프로젝트 문서1/배포 관련/배포 사이트 정보.html의 라이브 배포 현황을 기준으로 정리했다.

5. EC2_SSH_KEY 넣는 법

EC2_SSH_KEY는 .pem 파일의 경로나 파일명이 아니라 파일 내용 전체다. 로컬에서 키 파일이 있는 위치를 찾은 뒤 아래처럼 내용을 열어 복사한다. 

cat /path/to/hibot-d-server-key.pem

Secret 칸에는 private key 파일의 첫 줄부터 마지막 줄까지 전체가 들어가야 한다.

private key 파일 본문 전체
주의: private key 본문은 절대 문서, 채팅, git commit에 남기지 않는다. GitHub Actions Secret 입력 화면에만 붙여 넣는다.

6. AWS 키 권한

배포 자동화용 AWS 키는 가능하면 전용 IAM User 또는 Role로 만들고 최소 권한만 준다. 현재 배포 방식 기준으로 필요한 권한은 프론트 S3 업로드와 CloudFront 캐시 무효화다.

권한 대상 이유
s3:ListBucket beauty-book-hair-front aws s3 sync --delete 실행
s3:GetObject, s3:PutObject, s3:DeleteObject beauty-book-hair-front/* 정적 빌드 산출물 업로드/삭제
cloudfront:CreateInvalidation E11NF3HMOB52NI 배포 후 CloudFront 캐시 무효화
임시로 기존 AWS 키를 쓸 수는 있지만, 권장 방식은 배포 전용 최소 권한 키다. 이미지 업로드용 앱 키와 배포 자동화 키를 분리하면 사고 범위가 줄어든다.

7. GitHub Actions에서 쓰는 방식

workflow 파일에서는 아래처럼 secrets.로 참조한다. 

env:
  AWS_REGION: ${{ secrets.AWS_REGION }}
  FRONT_S3_BUCKET: ${{ secrets.FRONT_S3_BUCKET }}
  CLOUDFRONT_DISTRIBUTION_ID: ${{ secrets.CLOUDFRONT_DISTRIBUTION_ID }}
  NEXT_PUBLIC_API_URL: ${{ secrets.NEXT_PUBLIC_API_URL }}
ssh -i ~/.ssh/deploy_key ${{ secrets.EC2_USER }}@${{ secrets.EC2_HOST }} \
  "sudo systemctl restart milla-server"

8. 이 값들이 필요한 이유

백엔드 배포

GitHub Actions가 빌드한 JAR를 EC2로 복사하고 milla-server.service를 재시작해야 한다. 그래서 EC2 접속 정보와 SSH private key가 필요하다.

프론트 배포

Next.js 정적 export 결과물을 S3 버킷에 동기화하고 CloudFront 캐시를 비워야 한다. 그래서 AWS 키, 버킷명, CloudFront distribution ID가 필요하다.

API URL 고정

프론트는 빌드 시 NEXT_PUBLIC_API_URL을 읽는다. 운영 빌드에 이 값이 없으면 로컬 백엔드 주소로 붙을 수 있다.

비밀값 분리

AWS 키와 SSH 키는 git에 들어가면 안 된다. GitHub Secrets에 넣으면 workflow에서는 사용할 수 있지만 로그에는 노출되지 않는다.
