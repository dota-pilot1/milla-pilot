배포 사이트 정보

milla 후원 플랫폼(서버+프론트)이 english-agent-hub의 EC2/도메인/CloudFront/S3 슬롯을 인계받아 배포됨. 공개 주소: https://dxline-tallent.com
⚠️ 비밀값(JWT/AWS/DB 비번)은 이 문서에 기재하지 않음 — 서버 .env에만 존재.

3. milla 배포 리소스 (EC2 내부 신규)

systemd 서비스milla-server.service (-Xmx512m -Xms256m)
백엔드 포트4301 (english-agent-hub와 동일 슬롯 → Nginx 무변경)
JAR / env/home/ubuntu/milla/app.jar · /home/ubuntu/milla/.env (600)
DBdonation_platform — 기존 english-agent-hub-postgres 컨테이너(:5436, postgres/postgres)에 DB만 추가
CORSSecurityConfig.java에 https://dxline-tallent.com 추가(재빌드 반영)
프론트 API빌드 시 NEXT_PUBLIC_API_URL=https://dxline-tallent.com
.env 항목JWT_SECRET · AWS_*(이미지) · MAIL_USERNAME/MAIL_PASSWORD(Gmail 앱비번, beauty-book /home/ubuntu/.env에서 재사용) — 값은 서버에만
동적 라우트정적 export 호환 위해 /donate/facility?id=·/donate/item?id= 쿼리스트링으로 전환
인계로 중단된 것: englishagenthub.service stop+disable (app.jar·DB 보존) · S3 프론트만 milla로 교체. 되돌리면 복구 가능(§6).

5. 재배포 명령

백엔드

cd donation-platform-server && ./gradlew build -x test
scp -i "<pem>" build/libs/donation-platform-server-0.0.1-SNAPSHOT.jar ubuntu@13.209.195.64:/home/ubuntu/milla/app.jar
ssh -i "<pem>" ubuntu@13.209.195.64 "sudo systemctl restart milla-server && sleep 3 && systemctl is-active milla-server"

프론트

cd donation-platform-front
NEXT_PUBLIC_API_URL=https://dxline-tallent.com npm run build     # → out/
aws s3 sync out/ s3://beauty-book-hair-front --delete --region ap-northeast-2
aws cloudfront create-invalidation --distribution-id E11NF3HMOB52NI --paths "/*"

상태 확인

curl -s -o /dev/null -w "%{http_code}\n" https://dxline-tallent.com/api/site-settings   # 200
ssh -i "<pem>" ubuntu@13.209.195.64 "systemctl status milla-server --no-pager; free -h"

6. 롤백 (english-agent-hub 복구)

# 백엔드 원복
ssh -i "<pem>" ubuntu@13.209.195.64 "
  sudo systemctl stop milla-server && sudo systemctl disable milla-server
  sudo systemctl enable englishagenthub && sudo systemctl start englishagenthub"
# 프론트 원복: english-agent-hub-front 를 다시 빌드해 같은 버킷에 sync + 무효화
주의: 프론트 S3는 milla로 --delete 동기화돼 english-agent-hub 파일이 지워졌으므로, 원복하려면 english-agent-hub 프론트를 재빌드해 다시 sync해야 한다(DB·컨테이너는 보존됨).

7. "정식 배포" 시 분리할 것 (지금은 쇼케이스)

  • milla 전용 도메인 (지금은 dxline-tallent.com 인계) + 전용 CloudFront/ACM
  • milla 전용 EC2/인스턴스 (지금은 beauty-book·english-agent-hub와 2GB 공유 — 메모리 빠듯)
  • milla 전용 DB 컨테이너 (지금은 english-agent-hub-postgres에 DB만 추가)
  • 이미지 업로드: 전용 S3 버킷 + IAM 키 (§ 별도) · CORS dxline-tallent.com PUT 허용
  • CORS 환경변수화 (현재 코드 하드코딩)
